# 工业温度监控系统 (Modbus RTU over TCP)

本系统是一个基于 WPF 开发的现代化工业温度与设备状态监控平台，专门针对多通道、多从站的 **Modbus RTU over TCP** 协议场景设计。具备动态配置、高并发轮询、简易/高级双视图无缝切换以及完全的自适应布局等特性。

## 🌟 核心特性

- **动态多通道扩展**：不需要硬编码修改，在 UI 或 `settings.json` 即可无限增加或移除 TCP 通道（如串口服务器的不同端口）。
- **多从站并发轮询**：每条 TCP 通道自动接管并异步轮询其下挂载的所有从站设备，最大化总线利用率。
- **配置热加载与持久化**：所有的通信配置（IP、端口、从站 ID）随时可以在 UI 保存并固化至本地 `settings.json`。
- **简易/调试 双视图分离**：
  - **默认简易模式**：专为操作监控人员设计，自动隐藏冗余参数，仅显示醒目的核心温度与报警提示，卡片全屏等比自适应。
  - **高级调试模式**：一键切出串口配置面板、通信帧级日志、读写耗时统计及单片机内部寄存器读写控制台，专供实施与维保。
- **现代化深色工业 UI**：采用基于 WPF 的深色扁平/微拟物结合设计风格，状态指示灯与温控自适应变色反馈。

## 🚀 快速启动

1. 确保安装了 **.NET 8.0 Desktop Runtime**。
2. 运行 `ModbusMonitor.exe`。
3. 如果目录下不存在 `settings.json`，程序会自动生成包含单通道设备的默认模板。

## ⚙️ 配置文件 (settings.json)

配置会在修改后自动同步至此文件，您可以手动编辑它：
```json
{
  "Channels": [
    {
      "Name": "COM1",
      "Ip": "192.168.1.80",
      "Port": 10001,
      "Slaves": [1, 2]
    },
    {
      "Name": "COM2",
      "Ip": "192.168.1.80",
      "Port": 10002,
      "Slaves": [1]
    }
  ]
}
```

## 🛠️ 测点（Modbus）协议映射

寄存器（Holding Registers）采用标准 03(读) / 06(写) 功能码，对应功能如下：

| 地址偏移 | 读/写 | 数据类型 | 含义 |
|---------|------|---------|------|
| 40001   | Read | Float/Word | 实际温度 (带一位小数) |
| 40002   | R/W  | Word | 预警温度阈值 |
| 40003   | R/W  | Word | 报警温度阈值 |
| 40004   | Read | Enum | 报警指示 (0=正常, 1=报警, 2=超高) |
| 40005   | Read | Enum | 故障指示 (0=正常, 1=温度故障, 2=电池低) |
| 40006   | Read | Float/Word | 电池电压 |
| ...     | ...  | ... | （其它含单片机ID、光感档位等详见源码手册）|

## 🏗️ 架构说明

- **UI 层 (WPF / MVVM)**：依靠 `MainWindow`(绑定 `MainViewModel`) 使用 `ItemsControl` 与 `UniformGrid` 并发映射所有的 `ChannelViewModel` 和拉平合并的 `DeviceViewModel` 集合。
- **数据层 (Models)**：使用 `DeviceData` 保存纯净的运行时属性（实现 `INotifyPropertyChanged`）。
- **通信引擎 (Services/ModbusService.cs)**：基于原生 `NetworkStream` 实现 TCP 帧包装 RTU Payload 结构。每个通道单开线程闭环处理超时与重连重试。
